#추후 부족한 부분 수정 예정
#우선순위 표시 (1~5)

- API 구성
	유저 정보 관련 API(CRUD)
		CREATE - (5)유저 회원가입 페이지에서 유저의 요청을 받아 DB USER 테이블에 정보를 작성할 수 있는 API
				- 기본적인 유저 테이블 정보와 휴대전화 문자 인증과정 중 인증번호 랜덤생성 및 유효성 검증과정을 포함한다.
			- (5)로그인 페이지에서 로그인을 요청 할 경우 DB USER 테이블에서 인증 여부 판정 후 인증 토큰과 정보를 제공할 수 있는 API(보안 문제로 POST 요청)
			- (3)로그아웃 요청 시에 인증 토큰을 무효화 시킬 수 있는 API
		READ - (5)유저의 홈 화면 및 로그인 페이지에서 로그인 인증이 존재할 경우 DB USER 테이블에 정보를 제공할 수 있는 API
			- (2)아이디 찾기 요청 시에 인증과정에 일치하는 유저 정보를 필터링한 후에 DB USER 테이블에 정보를 제공할 수 있는 API
				- 이메일 또는 전화번호를 통한 인증과정 포함
		UPDATE - (3)유저의 개인정보 수정 페이지에서 유저의 요청을 받아 DB USER 테이블에 정보를 수정할 수 있는 API
			- (2)비밀번호 찾기에서 유저의 요청을 받아 DB USER 테이블에 비밀번호 정보를 수정할 수 있는 API
				- 이메일 또는 전화번호를 통한 인증과정 포함
		DELETE - (1)유저의 회원탈퇴 시 DB USER 테이블에 정보를 삭제할 수 있는 API
	차량 정보 관련 API(CRUD)
		CREATE - (5)유저 회원가입 페이지나 차량 등록 페이지에서 유저의 요청을 받아 DB Car_Info 테이블에 정보를 작성할 수 있는 API
		READ - (2)차량 등록/ 수정 페이지에서 DB Car_Info 테이블에 정보를 제공할 수 있는 API
		UPDATE - (2)차량 수정 페이지에서 유저의 요청을 받아 DB Car_Info 테이블에 정보를 수정할 수 있는 API
		DELETE - (1)차량 정보 삭제시 DB Car_Info 테이블에 정보를 삭제할 수 있는 API
	예약 정보 관련 API(CR)
		CREATE - (5)호출하기 버튼 선택 시 일련의 과정에서 차량 정보와 주차장 내부 위치, 주차장 정보, 충전시간 정보를 입력 받아 DB Working_Schedule 테이블을 작성할 수 있는 API
		READ - (3)이용현황 및 홈화면에서 DB Working_Schedule 테이블에 User_Id 기준으로 필터 후 정보를 제공할 수 있는 API

	로봇 정보 관련 API(RU)
		UPDATE - (5)로봇 정보 테이블에 MQTT를 통해 라즈베리 파이로 로봇 상태를 전송 받아 특정 상황 별 True, False 판정 후 DB Robot_Info 테이블의 state와 DB Working_Schedule 테이블의 Is_Finished 정보를 수정할 수 있는 API
		// READ - 관리자 API(후순위) 
	로봇 로그 관련 API(CR)
		CREATE -  (3)로봇 로그 테이블에 MQTT를 통해 라즈베리 파이로 로봇 상태, 로봇 위치를 전송 받아 DB Robot_Log 테이블에 정보를 작성할 수 있는 API
		// READ - 관리자 API(후순위)
	주차장의 위치 관련 API(R)
		베이스 정보 - SQL문으로 DB에 직접 입력(수정과 확장성을 고려하여 추후에 CU구현 예정)
		READ - (5)예약 선택 화면에서 주차장 선택, 주차장위치 선택 페이지에서 주차장 이미지와 주차장 정보를 사용자 GPS정보 또는 검색 받은 위치의 GPS를 바탕으로 가까운 순서로 정렬 후 DB Parking_Area_Info 테이블에서 정보를 제공할 수 있는 API 
	주차장 내부 위치 관련 API(R)
		베이스 정보 - SQL문으로 DB에 직접 입력(수정과 확장성을 고려하여 추후에 CU구현 예정)
		POST - (5)로봇 정보 테이블에 State가 충전 받을 수 있는 상태로 변경 시에, 로봇 정보 테이블의 주차장 정보 테이블 PK를 기준으로 필터링한 후에, 충전 종료 여부(Is_Finished)가 false인 예약 중 가장 빠른 순서의 예약 테이블 PK에 해당하는 Parking_Codiante_Info_Id 정보를 내부 위치 인덱스를 제외하고, MQTT를 통해 라즈베리 파이로 POST요청을 보낼 수 있는 API 
				- 또는 로봇 정보 UPDATE API 동작 과정 중 직접 전송 
	기능 별 추가 구현 사항
		알림 기능 (Firebase사용)
			POST - (2)유저 테이블 또는 유저 테이블을 참조하고 있는 차량 정보 테이블, 예약 정보 테이블의 생성, 수정, 삭제 과정이 이루어진다면, 알림을 전송  
	외부 API
		- 네이버 지역 검색 API(2)
			- 주차장 정보 선택 페이지에서 GPS 허용 동의를 받지 못했을 경우, 사용자가 직접 검색을 수행하여, 주차장을 선택할 수 있는 API
			- 검색 결과는 랜더링하지 않고, 가장 상단검색 결과의 GPS 좌표만 추출하여, 주차장의 위치 API에 활용
		- 네이버 클라우드 SMS API(2)
			- 전화번호 인증(비밀번호, 아이디, 회원가입 등)에 활용
		- GMAIL 또는 네이버 메일 POP/IMAP(2)
			- 이메일 인증(비밀번호, 아이디, 회원가입 등)에 활용
		- 추후) 카카오 또는 네이버 지도 API 활용
			- 지도 랜더링 후 DB 상 GPS 좌표 기준으로 충전소를 마커로 표시
